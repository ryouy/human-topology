# 仕様書（perDistMap）

## 1. 目的

日本語 Wikipedia 上の**日本人人物ページのみ**で構成されたリンクグラフを構築し、人物間の**最短リンク距離（人物ページのみを経由）**を可視化する Web アプリである。

現実の人間関係ではなく、**記事間リンク構造上の知識空間**を表す。

## 2. ビュー

### 2.1 全体マップ

- ネットワーク全体の俯瞰
- ズーム・パン（2D）／回転・ズーム・パン（3D）
- 人物検索
- ノードサイズの切替（被リンク数 / degree / betweenness）
- ノードクリックで詳細パネル（オンデマンド取得）
- 2D / 3D 切替

### 2.2 個人起点マップ

- 選択人物を中心に、1〜3 ホップ（可変）の近傍のみ表示
- ノードクリックで「再起点」または詳細表示（同一コンポーネントで実装）
- 2D / 3D 切替、検索・詳細は全体マップと同様

## 3. データ定義（要約）

- **ノード**: 日本語 Wikipedia の日本人**人物**記事のみ
- **エッジ**: 人物記事から人物記事への**wikilink のみ**（有向）
- **距離**: 人物ノードのみを通る最短経路のリンク数

非人物ページはノードにも経由点にも使わない。詳細は [data-definition.md](./data-definition.md) および [exclusion-rules.md](./exclusion-rules.md)。

## 4. graph.json の方針

- 描画・検索に必要な**最小メタデータ**のみ
- **要約（summary）は全件を埋め込まない**
- クリック時に Wikipedia API から要約・サムネイル等を取得し、フロントでメモリキャッシュする

## 5. 技術スタック

| 領域 | 技術 |
|------|------|
| 前処理 | Python 3、networkx、requests |
| フロント | Next.js（App Router）、TypeScript、Tailwind CSS |
| 2D グラフ | react-force-graph-2d |
| 3D グラフ | react-force-graph-3d |

## 6. データ収集の二段構え

1. **Phase 1（MVP）**: MediaWiki / Wikimedia **API** でシードから拡張し `graph.json` を生成
2. **Phase 2（将来）**: Wikimedia **ダンプ**（page, page_props, redirect, pagelinks, categorylinks）へ移行可能な取得層の抽象化

分類・グラフ構築・指標・JSON 出力は取得元に依存しないモジュールとして実装する。

## 7. 受け入れ条件（MVP）

- 人物ページのみのグラフが生成される
- 非人物ページが距離計算に混ざらない
- `graph.json` は軽量で summary 全件なし
- 全体マップ・個人起点マップが 2D で利用可能
- 2D / 3D 切替可能
- ノードクリックで Wikipedia 由来の基本情報表示＋メトリクス表示
- 同一人物の再取得を避けるキャッシュ
- README の手順でローカル起動可能
